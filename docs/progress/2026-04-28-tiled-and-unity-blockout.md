### Tiled Map Editor 採用とレイアウト作成

- レイアウトデータの **真実の源** をどこに置くか議論
- 業界標準ツール **Tiled Map Editor**(無料、オープンソース)を採用
- itch.io から Windows 版をダウンロード&インストール
- 100×80 タイル(1タイル=1m=32px)のマップを作成

    - レイヤー: walls / water / houses / crop_fields

- 壁と水路を全て 2m 厚/幅に統一、隙間をゼロに
- 民家を 10×10m × 4軒、畑を 16×20m × 2枚に統一

### Tiled → Unity 自動配置の実装

- `.tmj` (JSON) ファイルを `unity/VRC-FantasyWorld-Unity/Assets/Maps/` に配置
- C# エディタスクリプト `Assets/Editor/VillageImporter.cs` を作成
    - `JsonUtility` で TMJ をパース
    - 各オブジェクトを `GameObject.CreatePrimitive(Cube)` に変換
    - レイヤーごとにマテリアル色を割り当て(壁=茶、水=青、家=クリーム茶、畑=緑)
    - 高さは custom property `height_m` から、無ければレイヤーごとのデフォルト
- メニュー **VRChat Village → Import from Tiled** からワンクリック実行
- メニュー **VRChat Village → Clear Village** で削除も可
- **初回実装で Y軸反転バグ**:Tiled の Y(下向き) → Unity の Z(奥向き)で反転を忘れていた → 修正

## 決まったこと

- **配置の真実の源は Tiled (.tmj)** 、 Unity は表示先
- **`.tmj` は unity/Assets/Maps/ に置く**(unity/ は git管理外なので、レイアウト履歴は git で追わない)
- **編集ループ**: Tiled で編集 → Unity で Import → 確認 → Tiled で修正 → 再 Import
- **座標系**: 1 タイル = 1m = 32px、Tiled Y下向き → Unity Z奥向き(反転)
- Novel の暴力的な描写を含むファイルは VR 制作には参照しない
- ロゴ・アイコンは AI生成より自作 SVG プレースホルダーが安全

## 詰まったポイントとその対処

- **Edit ツールが日本語ファイルで途中切れする** → bash heredoc で全体書き直し(複数回適用)
- **Tiled の「色」が2種類あって混乱**(タイントカラー vs オブジェクトレイヤー色) → オブジェクトレイヤー節の「色」が正解
- **壁の厚みを表現したい** → 矩形1個ではなく4本の壁(N/S/W/E)で囲む方式を採用
- **Y軸の向き間違い** → 正しくは `unity_z = mapHeightM - tiled_y - h/2`
- **Anthropic 分類器が王宮プロット情報で反応** → CLAUDE.md に「読まないファイル」リスト明記、新セッションで対応
