    2026-04-28: Tiled 採用 + 辺境の村レイアウト + Unity 自動ブロックアウト¶

    やったこと¶

    Novel との連携と戦略立案¶

    • Novel フォルダ(C:\Users\hoeho\Documents\Claude\Novel)を VR 制作の参照源として接続
    • 308 ファイル / 13 カテゴリの設定資料を全体把握
    • VR 化に直結するカテゴリ(地理・国家・建築)を特定、戦略を立案
    • 暴力描写を含む 12_国家/東の国/王宮の構造（暫定）.md 等を 読まないファイル として CLAUDE.md に明記
      • Anthropic の自動分類器を不必要に刺激しないため
      • 「読まないリスト」と「読んで良いリスト」を整理

    第1ワールド「辺境の村」の詳細設計¶

    • VR 化の最初の対象を 辺境の村 に決定(王宮ではなく汎用の入植村)
    • 詳細設計書 docs/world-spec/frontier-village.md を作成
    • VR 用世界観抜粋 docs/worldbuilding/excerpts/world-summary-for-vr.md 作成
    • 水回りの仕組み を組み込み、立体的な村に進化:
      • 川 → 取水路 → 術式石①(浄化) → 集落 → 下水池 → 術式石②(浄化) → 川下に戻す
      • 「川から借りた水は、きれいにして川に返す」という設計思想
    • 配置図 SVG を作成、複数回イテレーション

    Tiled Map Editor 採用とレイアウト作成¶

    • レイアウトデータの 真実の源 をどこに置くか議論
    • 業界標準ツール Tiled Map Editor(無料、オープンソース)を採用
    • itch.io から Windows 版をダウンロード&インストール
    • 100×80 タイル(1タイル=1m=32px)のマップを作成
    • オブジェクトレイヤー で防壁・水路・建物・畑を矩形で配置
      • レイヤー: walls / water / houses / crop_fields
    • ユーザーが配置 → Python スクリプトで整地(1mグリッドにスナップ、クラス名 typo 修正)
    • 壁と水路を全て 2m 厚/幅に統一、隙間をゼロに
    • 民家を 10×10m × 4軒、畑を 16×20m × 2枚に統一

    Tiled → Unity 自動配置の実装¶

    • .tmj (JSON) ファイルを unity/VRC-FantasyWorld-Unity/Assets/Maps/ に配置
    • C# エディタスクリプト Assets/Editor/VillageImporter.cs を作成
      • JsonUtility で TMJ をパース
      • 各オブジェクトを GameObject.CreatePrimitive(Cube) に変換
      • レイヤーごとにマテリアル色を割り当て(壁=茶、水=青、家=クリーム茶、畑=緑)
      • 高さは custom property height_m から、無ければレイヤーごとのデフォルト
    • メニュー VRChat Village → Import from Tiled からワンクリック実行
    • メニュー VRChat Village → Clear Village で削除も可
    • 初回実装で Y軸反転バグ:Tiled の Y(下向き) → Unity の Z(奥向き)で反転を忘れていた → 修正

    結果¶

    • Tiled で編集した村レイアウトが、ボタン1つで Unity シーンに 約 13 オブジェクトの Cube として再現 される
    • ClientSim で歩いて確認可能
    • 「データ駆動で村を構築」というワークフローが実証された

    決まったこと¶

    • 配置の真実の源は Tiled (.tmj) 、 Unity は表示先
    • .tmj は unity/Assets/Maps/ に置く(unity/ は git管理外なので、レイアウト履歴は git で追わない)
    • 編集ループ: Tiled で編集 → Unity で Import → 確認 → Tiled で修正 → 再 Import
    • 座標系: 1 タイル = 1m = 32px、Tiled Y下向き → Unity Z奥向き(反転)
    • Novel の暴力的な描写を含むファイルは VR 制作には参照しない
    • ロゴ・アイコンは AI生成より自作 SVG プレースホルダーが安全

    詰まったポイントとその対処¶

    • Edit ツールが日本語ファイルで途中切れする → bash heredoc で全体書き直し(複数回適用)
    • Tiled の「色」が2種類あって混乱(タイントカラー vs オブジェクトレイヤー色) → オブジェクトレイヤー節の「色」が正解
    • 壁の厚みを表現したい → 矩形1個ではなく4本の壁(N/S/W/E)で囲む方式を採用
    • Y軸の向き間違い → 正しくは unity_z = mapHeightM - tiled_y - h/2
    • Anthropic 分類器が王宮プロット情報で反応 → CLAUDE.md に「読まないファイル」リスト明記、新セッションで対応

    次にやること¶

    • Unity に取り込んだブロックアウトを ClientSim で歩いて確認
    • スポーン位置を村の門付近に調整
    • Tiled に 門・術式石・スポーン地点・見張り台 を追加配置
    • 再 Import で反映確認
    • フェーズ2(環境構築)へ:HDRI Skybox(黄昏)、Terrain、Fog の設定

    雑感¶

    「世界観を Tiled でデザイン → Unity で自動ブロックアウト」というデータ駆動なワークフローが今日完成した。これは、世界観を変えたいときに Tiled を編集するだけで Unity 側が即時に追従するということで、 設計と実装の距離がほぼゼロ になった。

    Novel フォルダの参照を始めたら、自分が思っていた以上に世界観の量と質があった。これだけ作り込まれた設定を VR で歩けるようにする価値は大きい。

    ChatGPT のロゴ事件、Y軸反転の間違い、Edit の途中切れ、王宮プロットによる分類器反応など、 全工程に小さな罠が散りばめられていた。 CLAUDE.md にこれらを記録しているので、次のセッションでは同じ罠を踏まずに済む。

    本日の追加作業(同日後半: 環境構築フェーズ着手)¶

    Terrain 作成と地面の整備¶

    • Unity の Terrain で 村の地面と外側 を作成
    • サイズ: 200×200m(村+周囲50mのバッファ)、位置 (-50, 0, -50)
    • Terrain Layers でテクスチャ適用:
      • Diffuse + Normal Map の組み合わせを Layer に設定
      • 1m=32px の単位は Tiled と Unity 共通で揃え
    • レイヤー概念の整理:
      • Layer 0(リスト一番上)= ベース、地面全体に自動適用
      • Layer 1, 2... = ブラシで重ね塗り
      • 「Create Layer」=テクスチャ画像から新規作成、「Add Layer」=既存 .terrainlayer を追加

    Stylized Trees パック導入¶

    • Stylized Trees - Cartoon Lowpoly Mobile Pack Free を Asset Store から取得
    • Mobile 最適化版なので VRChat 軽量、Built-in Render Pipeline 互換
    • Paint Trees で配置 → 大きいブラシ + Shift で全消去できる

    Skybox 検討中¶

    • AllSky Free をダウンロード、Import 時に黄昏系を選別する方針
    • 全部 Import すると重いので、Asset Store ページのプレビューで黄昏系1〜2個に絞る予定

    ビルドサイズ確認¶

    • Terrain あり: 約 100MB
    • Terrain なし: 約 70MB
    • → Terrain が約 30MB(妥当範囲、軽量化余地あり)
    • PC ワールドとしては問題なし、目標 50MB は完成形に向けての軽量化で対応

    学んだこと(操作 Tips)¶

    • Terrain ペイントブラシの dropdown は「Stamp Terrain」じゃなくて「Raise or Lower Terrain」を選ぶ
    • レイヤー色の設定は「オブジェクトレイヤー」セクションの「色」(Tint Color じゃない)
    • Tiled は無料の業界標準、TMJ は JSON でテキスト編集可能
    • .tmj ファイルは Unity Assets/Maps/ に置けば Unity から読める
